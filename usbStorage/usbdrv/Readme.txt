Эта папка содержит только USB firmware. Скопируйте её как есть в Ваш собственный проект и добавьте все .c and .S файлы в Ваш проект (эти файлы помечены звёздочками в списке ниже). Также скопируйте usbconfig-prototype.h как usbconfig.h в Ваш проект и отредактируйте его в соответствии с Вашей конфигурацией.


ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ
========================
Техническая документация (API) для firmware драйвера содержится файл в файле "usbdrv.h". Пожалуйста тщательно прочитайте его! Опции конфигурироважния задокументированы в "usbconfig-prototype.h".

Драйвер состоит из следующих файлов:
  Readme.txt ............. файл, который Вы сейчас читаете.
  Changelog.txt .......... информация по версиям релизов драйвера.
  usbdrv.h ............... установки интерфейса драйвера и технические документы.
* usbdrv.c ............... часть драйвера, написанная на языке высокого уровня. Прилинкуйте этот модуль к Вашему коду!
* usbdrvasm.S ............ ассемблерная часть драйвера. Этот модуль в основном является заглушкой, который включает 
                           один из файлов usbdrvasm*.S в зависимости от тактовой частоты процессора.
                           Прилинкуйте этот модуль к Вашему коду!
  usbdrvasm*.inc ......... ассемблерные процедуры для разных тактовых частот. Эти файлы подключаются в 
                           usbdrvasm.S, не линкуйте их напрямую!
  asmcommon.inc .......... Общие ассемблерные процедуры. Подключается в usbdrvasm*.inc, не линкуйте его напрямую!
  usbconfig-prototype.h .. Прототип для Вашего собственного файла usbdrv.h.
* oddebug.c .............. Функции отладки. Работают только тогда, когда определен DEBUG_LEVEL (оператором #define)
                           с величиной больше 0. Линкуйте этот модуль к Вашему коду!
  oddebug.h .............. Определения интерфейса для модуля отладки.
  iarcompat.h ............ Определения для совместимости с C-компилятором IAR.
  usbdrvasm.asm .......... Блок совместимости  с C-компилятором IAR. Используйте этот модуль вместо usbdrvasm.S,
                           когда Вы ассемблируете с средой отладки IAR.
  License.txt ............ Лицензия Open Source для этого драйвера.
  CommercialLicense.txt .. Дополнителльная коммерческая лицензия для этого драйвера.
  USBID-License.txt ...... Постановления и условия для использования частных ID
                           значений USB для специфических целей.

(*) ... Помеченные звездочкой файлы должны быть прилинкованы в Вашему проекту.


ТАКТОВАЯ ЧАСТОТА ЯДРА МИКРОКОНТРОЛЛЕРА
======================================
Мы предоставляем ассемблерные модули для тактовых частот 12, 15, 16 и 16.5 МГц. Другие тактовые частоты не поддерживаются. Действующая тактовая частота должна быть сконфигурирована в usbdrv.h, если Вы не используете частоту по умолчанию 12 МГц.

12 МГц
Это традиционная часота AVR-USB, потому что она минимальная, которая может удовлетворить ограничениям спецификации USB.

15 МГц
То же самое, как и 12 МГц, но добавлено для задержки несколько команд NOP. С другой стороны, высшая тактовая частота позволяет ввести некоторые циклы, которые могут уменьшить результирующий размер кода по сравнению с версией 12 МГц. 

16 МГц
Эта тактовая частота добавлена для пользователей платы Arduino и других отладочных плат, которые поставляются с впаянным кристаллом 16 МГц. Такая частота может также потребоваться и для Вас по соображениям быстродействия. Поскольку частота 16 МГц не делится на частоту бит 1.5 МГц низкоскоростного USB, в этом случае код получается несколько изощренным, и приходится вставить цикл каждый третий байт.

16.5 МГц
Ассемблерный модуль на этой частоте отличается от других модулей, потому что он основывается на встроенном RC генераторе тактов микроконтроллера, который имеет точность только 1%. Код приемника имеет циклы с переходами для компенсации девиаций тактовой частоты. Точности в 1% можно достичь путем калибровки внутреннего генератора RC AVR. Пожалуйста имейте в виду, что можно использовать только модели AVR с внутренним PLL-генератором (PLL - Phase-Locked Loop, фазовая автоподстройка частоты, ФАПЧ) 64 МГц, поскольку RC-генератор 8 МГц не может быть настроен на 16.5 МГц. Это включает очень популярные серии ATTiny25, ATTiny45, ATTiny85, а также ATTiny26.

См. пример EasyLogger на http://www.obdev.at/avrusb/easylogger.html для кода калибровки RC-генератора, основываясь на тактах фреймов USB.

20 МГц
Этот модуль для тех, которые не желает использовать что-либо кроме как по максимуму. Поскольку 20 МГц не делится на частоту бит 1.5 МГц низкоскоростного USB, код использует те же приемы, что и в модуле на 16 МГц.


ИДЕНТИФИКАТОРЫ USB (USB IDENTIFIERS)
====================================
Каждое устройство USB нуждается в идентификаторах вендора и продукта (VID и PID). VID получают от usb.org по цене в 1500 USD. Как только вы получили VID, Вы можете назначать PID-ы по желанию.

Так как цена в 1500 USD слишком высока для маленьких компаний и увлеченных хобби, мы предоставляем некоторые пары VID/PID бесполатно. Для деталей см. файл USBID-License.txt.

Objective Development также имеет некоторые лицензионные предложения, которые включают ID продукта. Детали см. на http://www.obdev.at/avrusb/.


СИСТЕМА РАЗРАБОТКИ
==================
Этот драйвер разработан и оптимизирован под компилятор GNU версии 3 (gcc 3). Он работает также с компилятором gcc 4, но при этом код получается бОльшего размера. Мы рекомендуем использовать компилятор GNU, поскольку он свободно доступен. AVR-USB также портирован под компилятор и ассемблер IAR. Проект протестирован под IAR 4.10B/W32 и 4.12A/W32 на ATmega8 с моделями памяти "small" и "tiny". Протестированы не все релизы вместе с IAR CC, и таким образом драйвер под IAR может компилироваться с ошибками. Пожалуйста имейте в виду, что gcc более эффективен для usbdrv.c, поскольку этот модуль был сознательно оптимизирован для gcc.
